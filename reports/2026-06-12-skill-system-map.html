Operating Model

Three layers

OpenAI system skills provide built-in capabilities. Portable repo skills live in ~/src/codex-skills/skills and are shared across Codex, Agents, Cursor, and Claude. Private local skills live in ~/.codex/skills and ~/.agents/skills when the workflow is personal, customer-specific, Mira-specific, or machine-local.

One workflow spine

vincent-workflow owns durable task state, decisions, blockers, handoffs, and cleanup obligations. Specialist skills own their domain work. Reports, closeouts, and local-brain memory each have their own owner.

Workflow Spine

The core design is simple: a specialist skill does the work, then shared workflow owners handle durable coordination. That avoids duplicate ledgers and keeps future sessions discoverable.

1. Find or trigger skill

find-skills, skill descriptions, and repo guidance route the task.

2. Specialist acts

Domain skills handle CI, support, frontend, observability, secrets, Velora, or other focused work.

3. Workflow state

vincent-workflow captures durable decisions, blockers, handoffs, known issues, and cleanup.

4. Proof surface

codex-html-report creates durable reader artifacts. Runtime proof stays in the owning runtime or domain tool.

5. Closeout

codex-closeout shapes the final chat response with verification, commit/push, cleanup, and real residual risk.

First-principles check: the workflow now asks what goal, facts, assumptions, constraints, simplest correct solution, proof, and revisit signal matter before a decision becomes durable.

Creation And Validation Gates

New skill creation

New skills that mention decisions, blockers, handoffs, closeout, reports, or memory must include a Workflow Coordination section. The standard section comes from python3 scripts/check-workflow-links.py --template.

All-skill enforcement

bash scripts/check-drift.sh runs scripts/check-workflow-links.py --all. GitHub Actions runs the same audit, installs the skills into runner surfaces, then runs the drift check.

Private skill path

Private skills mirror from ~/.codex/skills/<name> to ~/.agents/skills/<name>, stay in both manifests, and use sync-installed-extras.sh --check for drift proof.

Promotion rule

Generic, portable behavior belongs in this repo. Vincent/Mira/Velora/Selene identity, customer support specifics, secrets, and machine-local workflows stay local unless explicitly promoted.
