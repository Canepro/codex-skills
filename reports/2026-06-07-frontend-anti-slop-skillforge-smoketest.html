Done

Running the new flow over the SkillForge dashboard surfaced concrete problems on the first pass: off-system spacing, mixed radii, glow shadows and gradient shells used as default styling, and sixteen panels competing for equal visual weight. Subtraction first

Three things to remove or merge before adding

The new rule forces a subtraction list before any addition. On this dashboard the candidates were obvious within a minute of reading:

  1. Merge "Managed score history" into the Leaderboard. It duplicates the same skills, scores, and deltas with worse density. The decision column is the only new signal and could be a Leaderboard sub-column.
  2. Move "Decision Legend" to a tooltip on first pill hover, or a help drawer. Pills repeat across the page; explaining each takes a full panel of permanent real estate.
  3. Drop "Unscored Skills" from this view. The panel is currently empty for managed skills and shows only a footnote paragraph. It belongs in a Benchmarks page, not in Control Room.

Skill assessment

Did each new rule pull its weight

Pulled its weight

Pre-Design Discovery surfaced spacing and radius chaos within the first minute. Subtraction First produced three concrete cuts. P0/P1/P2 visual severity made triage trivial. Concrete Defaults gave specific numbers to point to. The new Routing block at the top stopped scope creep into copy and perf.

Did not fire this time

Copy rules barely applied: the dashboard is already restrained, no greeting filler, no mood empty states. Done When applies only to implement mode, so it sat out. Most bad-to-good library entries target slop-heavier UIs.

Gaps spotted

The skill flags "equal visual weight" but does not give a count rule. Sixteen panels per screen is the central problem here and there is no explicit "if more than 6 to 8 panels, demote" guidance. The icon-on-everything pattern is P2 but probably under-weighted.

Net: the revisions transform the skill from a critique list into operating instructions. Worth adopting. Two small follow-ups noted below.

  • Add a panel-count rule to frontend-anti-slop/SKILL.md under Structure: "If a screen has more than 6 to 8 top-level panels, identify a primary three and demote the rest." Single sentence, high leverage.
  • Promote the icon-density rule from P2 to P1 in the Visual block. Icons on every nav row, KPI corner, panel head, and brand mark is a consistent slop tell.
