Findings

Activation language was too narrow

naming-quality and grill-me were useful but depended on the user phrasing the request in near-perfect skill terms. Natural prompts like "what should we call this" or "what am I missing" needed to be explicit triggers.

Routing table missed practical utilities

find-skills lacked direct routes for naming, pressure-testing, interface design, architecture improvement, document/PDF/media work, security utilities, Azure, and migration helpers.

Regression guard was incomplete

The trigger matrix did not cover 27 repo-managed skills before the activation pass. That made neglect easy: a skill could remain installed but invisible to fresh-session routing checks.
